## Cross-Reference Tema

### 🔧 Smart Tools → Agent (evolusi konsep inti)
Konsep inti course ini: Tools itu kaku (repetitif doang, no memory), Agent itu bisa belajar (ada memory + feedback).
- [[Course 3 Build your 1st Smart Tools]] — definisi Smart Tools: 4 komponen (trigger, source data, processing, delivery), TANPA memori.
- [[Course 5 Build your first AI Agent]] — upgrade Tools → Agent: tambahin memory loop + feedback system jadi 3 komponen inti agent.
- [[Course 8 Loop Engineering]] — loop = pola generik di balik Tools & Agent: 5 bagian (pemicu, memori, aturan, alat bantu, output).
#smart-tools #agent

### 🔁 Memory / Feedback Loop
Konsep "memori" muncul di 3 modul beda konteks — dari memori si tool/agent, sampai memori builder-nya sendiri.
- [[Course 5 Build your first AI Agent]] — memory loop bikin Agent bisa "inget" pengalaman & improve output-nya (kayak Ken, yang nyimpen konten rating 4 ke memori).
- [[Course 7 Build your 2nd Brain]] — 2nd Brain = memori buat BUILDER-nya (bukan agent-nya): nyimpen semua sesi building & diskusi biar AI bisa diakses ulang.
- [[Course 8 Loop Engineering]] — "memori" jadi salah satu dari 5 komponen struktur loop.
#memory #feedback-loop

### 🎛️ Orchestrator
- [[Course 6 Build your Orchestrator]] — modul utama: Orchestrator ngatur banyak agent/tools, punya 4 base logic (delegasi, tarik report, rangkum laporan).
- [[Course 7 Build your 2nd Brain]] — Orchestrator disebut lagi sebagai salah satu dari 3 cara buat "meng-Recall data" dari 2nd Brain (selain cloud chat & cloud code) — **direkomendasikan** sebagai cara utama akses.
- [[Course 8 Loop Engineering]] — Orchestrator posisinya di atas Agent dalam Corepath progression, sebelum masuk specialized tracks.
#orchestrator
